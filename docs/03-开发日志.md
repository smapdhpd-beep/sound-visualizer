## 2026-06-15（Chladni 核心逻辑修复）

### 完成项
- [x] Chladni 模式 `symmetry=1` 视觉效果重构
  - **问题**：无对称时粒子像散乱闪光点，只有调高对称阶才能看到清晰图案
  - **根因**：重生概率过高（~2%/帧）破坏节线连续性；频率瞬时跳变导致节线抖动；缺乏节点粘性；渲染光晕阈值过高
  - **修复措施**：
    1. **频率平滑**：新增 `smoothedBaseFreq` / `smoothedFreqSpread`，lerp 系数 0.15，节线稳定不抖动
    2. **重生概率降低 10 倍**：`0.98 → 0.998`（60fps 时 2% → 0.2%），静音时完全禁止重生
    3. **重生阈值归一化**：固定 `val < 0.018` 改为 `val / srcCount < 0.005`，适应 2~6 波源
    4. **节点粘性**：`nodeProx > 0.55` 时布朗运动最多降低 85%，阻尼提升最多 42.5%，粒子稳定在节线上
    5. **Onset 冲量削弱**：`onset * 500 → onset * 150`，以频率扰动为主，避免物理冲量打散图案
    6. **渲染增强**：强制 `lighter` 混合；所有粒子自带光晕（`glowSize = 2x~5x`，`glowAlpha = 0.03~0.25`）；默认粒子数 4000；拖尾 alpha 0.12
  - **验证**：node-canvas 离线模拟 800 帧，sym=1 平均亮度从 49 → 140，bright 像素比例从 26% → 53%，接近 sym=4 的 115/68%
- [x] 生成 40+ 组测试截图（`test-screenshots/`），覆盖 bass-heavy / high-vocal / full-mix / silent 多风格

### 收尾发现（关键！）
- **node-canvas 截图与浏览器实际观感存在系统性差异**：
  - Cairo CPU 合成 vs 浏览器 GPU Skia/CoreGraphics，颜色空间与 alpha 混合路径不同
  - 无 DPR 处理：Retina 屏上同样 `size=1.5` 的粒子会被渲染成 3×3 物理像素，截图里只是 1.5px
  - 快照 vs 连续视觉暂留：800 帧最终 PNG 无法还原人眼看到的流动拖尾感
  - 硬编码音频特征 vs 真实音乐频谱分布：测试脚本的 `bass=0.9` 不等于真实电子乐的 bass 曲线
- **结论**：离线测试只能验证算法逻辑和参数方向，最终效果必须以浏览器实机为准。明日工作流切换为「用户描述实机观感 → 我针对性调参」，不再依赖截图对比。
