# 08-Chladni 核心修复备忘

## 背景

用户反馈：Chladni 模式在 `symmetry=1`（无对称）时，画面只是一堆粒子跟随音频闪耀，完全看不到克拉尼图应有的几何图案；必须手动调高对称阶数（如 4）才能看到像样的效果。这与用户提供的 Processing 参考实现差距明显。

## 根因分析

通过 node-canvas 离线模拟 40+ 组参数对比，定位到 6 个相互叠加的问题：

| 问题 | 现象 | 根因 |
|------|------|------|
| 节线抖动 | 图案模糊、无法聚焦 | 波源频率每帧完全跟随音频瞬时特征，无平滑 |
| 重生过于频繁 | 已形成的线条被不断打断 | `rebirthProb ≈ 2%/帧`，4000 粒子 ≈ 80 个/帧被重置 |
| 缺乏节点粘性 | 粒子到达节线后很快被踢走 | 布朗运动恒定，无差别作用于所有粒子 |
| Onset 冲量过强 | 每次鼓点图案瞬间炸散 | `onsetKick = onset * 500`，粒子速度突变 |
| 渲染光晕阈值过高 | 节线太细，肉眼难辨 | 只有 `acc > 0.35` 的极少数粒子才有光晕 |
| Blend 模式不统一 | ink 主题下堆积感被抹平 | `source-over` 叠加半透明会让画面发灰 |

## 修复方案（已全部落地到 `js/renderers/chladniRenderer.js`）

### 1. 频率平滑（核心）
- 新增状态：`smoothedBaseFreq`、`smoothedFreqSpread`
- 每帧向目标值 lerp：`factor = 0.15`
- 效果：节线位置稳定，图案从"一盘散沙"到"成形"约需 3~5 秒，但成形后不再抖动

### 2. 重生概率降低 10 倍
- `Math.pow(0.98, dt*60)` → `Math.pow(0.998, dt*60)`
- 60fps 时从 ~2% 降到 ~0.2%
- 静音时（`activity ≤ 0.15`）完全禁止重生

### 3. 重生阈值归一化
- 固定 `val < 0.018` → `val / srcCount < 0.005`
- 适应用户通过 Tweakpane 调节的 2~6 波源数

### 4. 节点粘性（关键创新）
```
nodeProx = 1 - min(1, val / srcCount)
if nodeProx > 0.55:
    stick = (nodeProx - 0.55) / 0.45 * 0.85
    effBrownian *= (1 - stick)        // 最多降低 85%
    effDamp = 1 - (1-damp)*(1-stick*0.5)  // 阻尼最多提升 42.5%
```
- 粒子越接近深层节点，随机扰动越小、刹车越重
- 这是 sym=1 也能看到清晰线条的物理基础

### 5. Onset 处理策略调整
- 粒子物理冲量：`onset * 500` → `onset * 150`
- 主要扰动从"打散粒子"转向"扰动频率"：onset 时各波源频率 ×0.7~1.3
- 视觉上体现为"图案突变/重组"而非"炸开"

### 6. 渲染增强
- **强制 lighter 混合**：Chladni 模式忽略 theme.blend，统一使用 `lighter`，凸显堆积密度差异
- **全局光晕**：所有粒子都有光晕，与 `acc` 成正比
  - `glowSize = size * (2.0 + acc * 3.0)`  →  2x ~ 5x
  - `glowAlpha = alpha * (0.12 + acc * 0.25)`  →  低 acc 也有微弱光晕
- **默认粒子数**：2500 → 4000
- **拖尾 alpha**：0.22 → 0.12（更长拖尾，线条更连续）
- **粒子尺寸**：`1.0~4.2` → `1.5~6.0`
- **透明度**：`0.22~1.0` → `0.28~1.0`

## 验证数据

使用 node-canvas 模拟 800 帧（≈13.3 秒），对比 sym=1 与 sym=4：

| 版本 | 场景 | 平均亮度 | Bright > 80 比例 |
|------|------|---------|------------------|
| 修复前 | bass-heavy sym=1 | 49 | 26% |
| 修复前 | bass-heavy sym=4 | 115 | 68% |
| 修复后 | bass-heavy sym=1 | 140 | 53% |
| 修复后 | high-vocal sym=1 | 112 | 41% |
| 修复后 | silent sym=1 | 20 | 4% |

结论：sym=1 的可见度已接近修复前的 sym=4，且不同音乐风格（低音/高音/混合）产生明显不同的图案尺度。

## 参数映射（供后续调参与文档）

| 音频特征 | 控制对象 | 视觉效果 |
|----------|---------|---------|
| bass | 波源半径 `breathe = 0.30 + bass*0.30` | 低音强 → 波源外扩 → 大尺度简单图案（圆/三角） |
| high + centroid | 基频 `baseFreq` 与差异度 `freqSpread` | 高频强 → 高基频 + 高差异 → 复杂细碎图案（星形/多瓣） |
| overall | 推力 `strength` 与重生概率 | 总能量决定图案"凝聚度"，静音时完全散沙 |
| onset | 频率随机扰动 + 弱物理冲量 | 鼓点触发图案重组，保持节奏感 |

## 遗留待验证

1. **浏览器真机 FPS**：node-canvas 无法模拟 GPU 合成，4000 粒子 × lighter blend × 全局光晕在低端手机是否仍能 > 30fps 需实测
2. **模式切换闪屏**：本次未改动切换逻辑，但渲染器状态变化较大，需确认 destroy/init 是否干净
3. **ink 主题被覆盖**：Chladni 强制 lighter，若用户明确想要水墨低对比度效果，当前无法满足（如需支持，可后续加「强制 blend」开关）

## 离线验证的局限性（2026-06-15 收尾发现）

今天的验证全部基于 node-canvas（Cairo 后端），与浏览器实机存在以下系统性差异：

| 差异维度 | node-canvas | 浏览器 Canvas 2D |
|---------|-------------|------------------|
| 合成后端 | CPU 软件合成 | GPU 加速（Skia/CoreGraphics） |
| 颜色空间 | 线性数学运算 | sRGB + 浏览器色彩管理 |
| DPR 处理 | 无（1:1 像素） | 按 `devicePixelRatio` 缩放（Retina 为 2x/3x） |
| 视觉感知 | 单帧快照 | 连续视觉暂留 + 拖尾流动感 |
| 音频输入 | 硬编码数值 | 真实频谱分布 + 底噪 |

**结论**：离线测试只能验证「算法逻辑是否正确」「参数方向是否有效」，最终视觉效果必须以浏览器实机为准。明日工作流切换为「用户描述实机观感 → 针对性调参」。
