# 02-架构决策

记录项目中具有长期影响的技术选型与架构判断。

---

## AD-01：GUI 库选型 — Tweakpane

**决策**：MVP 阶段使用 Tweakpane 3.x（ESM CDN 引入）。

**理由**：
- 参数面板开箱即用，支持滑块、颜色、下拉等控件
- 可通过 CSS 自定义主题适配暗色界面
- 移动端通过底部抽屉封装解决占用屏幕问题

**约束**：
- 手机端默认折叠，仅保留右下角浮动按钮
- 面板从底部滑出，不遮挡中心视觉区

**替代方案**：自研极简面板（保留到 Phase 3 评估）。

---

## AD-02：Canvas 切换方案 — 双 Canvas 重叠显隐（历史保留）

**决策**：DOM 中同时存在两个绝对定位重叠 Canvas：
- `#canvas-2d`：专供 chladni 模式（当前唯一活跃渲染面）
- `#canvas-webgl`：历史保留，liquid 废弃后已隐藏

**理由**：
- 避免同一 Canvas 在 Safari 中动态切换 2D / WebGL 上下文导致上下文丢失
- 逻辑干净，各渲染器独占输出面，互不干扰
- 若未来需要 WebGL，可直接复用现有层，不污染 2D 链路

**当前状态（v1.0）**：
- 仅 `#canvas-2d` 可见，WebGL 层始终隐藏
- 两 Canvas `position: absolute; inset: 0; width: 100%; height: 100%`
- z-index 控制显隐，`pointer-events: none` 默认放行交互到顶层透明覆盖层
- 交互统一由 `#interaction-layer` 接收，再分发给当前激活的 Renderer

---

## AD-03：液态纹路模式 — 已废弃

**决策**：liquid 模式于 2026-06-17 正式废弃，不再维护。

**理由**：
- 用户明确决策：全力聚焦克拉尼单模式，液态与波形雕塑方向均放弃
- 文件 `js/renderers/liquidRenderer.js` 已标记废弃注释，App 中已移除注册
- `archive/dustRenderer.js` 已归档历史代码

---

## AD-04：配色系统 — 全局可选，不绑定模式

**决策**：5 套配色为全局状态，渲染器内自由切换。

| ID | 名称 | 感觉 |
|----|------|------|
| warmSand | 暖沙 | 真实克拉尼，沙粒在暗色金属板上 |
| neon | 霓虹 | 电子乐 club，高饱和青紫 |
| ink | 水墨 | 宣纸质感，墨点淡晕 |
| aurora | 极光 | 深空蓝绿，科技感 |
| audioDriven | 幻彩 | HSL 色相环漫游，onset 跳色，音频驱动 |

**理由**：
- 最大化用户创作自由度
- 与「成为自己的作品」产品目标一致

---

## AD-05：状态管理 — 纯原生 Class，无框架

**决策**：不使用 React/Vue/Zustand，用原生 ES Class + 事件驱动。

**理由**：
- 项目以 Canvas 渲染为核心，DOM 极少
- 避免框架运行时开销，保持 60fps
- 音频与渲染链路需要精确时序控制，命令式更直接
