## 0. 问题诊断：为什么 Dust 不受欢迎

当前 `DustRenderer` 的核心算法：
```
angle = sin(x*0.004 + time) * cos(y*0.004 + time*0.7) * 2π
force = (cos(angle), sin(angle)) * spread * (0.3 + bass*1.2)
```

**问题清单**：

| 问题 | 表现 | 根因 |
|------|------|------|
| **无结构** | 粒子像一锅粥在屏幕上乱飘 | 力场是简单的正弦乘积，没有吸引子/势阱 |
| **物理虚假** | 自称"呼吸光尘"但无呼吸感 | 没有真实的膨胀/收缩周期，也没有密度梯度 |
| **音频映射浅薄** | bass 只控制扩散幅度 | 中频/高频/节拍只提供随机冲击，无形态变化 |
| **对称生硬** | 镜像/旋转与力场无关 | 对称是后处理硬贴上去的，不是力场本身的对称 |
| **视觉脏** | 拖尾叠加后一片浑浊 | 高粒子数 + lighter 混合 + 低对比度 = 视觉噪音 |

**结论**：Dust 是一个"为了有第二个模式而存在的模式"，缺乏独立的视觉身份和算法灵魂。它既不像是某种物理现象，也不像是精心设计的数据可视化。

## 1. 克拉尼图的多形态扩展

### 1.1 数学背景：不同边界的波动方程

克拉尼图的本质是**二维亥姆霍兹方程** `∇²ψ + k²ψ = 0` 在特定边界条件下的本征模态。边界形状直接决定节点线（波节线）的几何图案。

当前实现：**圆形板（极坐标，软边界）**
```
ψ(r,θ) = sin(m·π·r) · cos(n·θ)
```

### 1.2 可扩展的边界形态

| 形态 | 数学表达 | 视觉特征 | 实现难度 |
|------|----------|----------|----------|
| **方形板** | `sin(mπx)·sin(nπy)` | 网格状节线，直角交叉，类似城市地图 | ★☆☆ |
| **矩形板** | `sin(mπx/a)·sin(nπy/b)` | 方形变体，长短边比例影响图案 | ★☆☆ |
| **环形板** | 贝塞尔函数组合 `Jₙ(α·r)·cos(nθ)` | 同心圆 + 放射线，像靶心或曼陀罗 | ★★☆ |
| **椭圆板** | 马丢函数 MathieuCE/CSE | 极度独特的"猫眼""四叶"变形图案，几乎没见过 | ★★★ |
| **正三角板** | 三角坐标系下的驻波 | 三向对称，六芒星衍生物 | ★★☆ |
| **六边形板** | 蜂窝状节线 | 天然与晶体/蜂巢美学契合 | ★★☆ |
| **圆扇形** | 扇形边界 + 周期角向条件 | 扇形切片内的放射图案 | ★★☆ |
| **双曲/抛物** | 双曲坐标下的波动方程 | 双曲线族节线，科幻感强烈 | ★★★ |

### 1.3 实施建议

**优先级 P1：方形板 + 环形板**
- 方形和圆形是最直观的对比。圆形给人"有机/自然"感，方形给人"数字/精密"感。
- 环形板只需在圆形基础上引入内孔半径参数 `r0`，将径向函数从 `sin(mπr)` 改为贝塞尔函数零点组合。
- 两者都可以复用现有的粒子系统 + 节点粘性算法。

**优先级 P2：椭圆板（差异化王牌）**
- 椭圆克拉尼图极其罕见，因为需要马丢函数。但可以用数值近似或预计算查表。
- 它的视觉语言独一无二："变形的对称"，既有圆形的流动感又有方形的结构感。
- 可以作为项目的"招牌形态"——别人做的都是圆/方，你做椭圆。

**实施方式**：
在 `ChladniRenderer` 中增加一个 `boundaryShape` 参数：
```javascript
const SHAPE_MODES = {
  circle:   (x,y,m,n) => polarWave(x,y,m,n),
  square:   (x,y,m,n) => cartesianWave(x,y,m,n),  // sin(mπx)·sin(nπy)
  annular:  (x,y,m,n) => besselWave(x,y,m,n),     // Jₙ(k·r)·cos(nθ)
  ellipse:  (x,y,m,n) => mathieuWave(x,y,m,n),    // 数值近似
};
```

所有形态共享同一套粒子更新逻辑，只替换 `_waveField` 函数。

## 2. 音频可视化的其他方向

### 2.1 方向分类矩阵

| 方向 | 代表形态 | 核心感知 | 适合音乐类型 |
|------|----------|----------|-------------|
| **几何驻波** | 克拉尼图（已有） | "沙粒在唱歌"、物理之美 | 电子/氛围/古典 |
| **时域波形** | 示波器、Lissajous | 音乐的"心电图"、raw | 电子/人声/噪音 |
| **频谱分析** | 频谱条、圆环频谱、瀑布 | 音乐的"解剖图"、信息密度高 | 所有类型 |
| **粒子物理** | 引力系统、发射器 | "能量的舞蹈"、混沌边缘 | 鼓点/Bass/摇滚 |
| **流体场论** | Navier-Stokes、反应扩散 | "液体在听音乐"、有机感 | 氛围/电子/爵士 |
| **轨迹艺术** | 波形堆叠、轨道环 | "时间的雕塑"、历史感 | 渐进/长曲/氛围 |

### 2.2 针对当前项目的具体提案

当前已有：
- A. 几何驻波（Chladni，强）
- B. 混沌粒子（Dust，弱）
- C. WebGL 流体占位（Liquid，未完工）

建议的新模式定位：

#### 提案一：频谱瀑布（Spectrum Waterfall）
- **概念**：每一帧的 FFT 频谱作为一条横向扫描线，历史帧向下滚动，形成 3D 瀑布效果。
- **为什么好**：
  - 经典永不过时，且音乐结构（drop/build/verse）一目了然
  - 与 Chladni 的"抽象几何"形成完美互补：一个偏物理模拟，一个偏数据可视化
  - 实现简单，性能稳定，不需要复杂物理
- **音频映射**：
  - 横轴 = 频率（对数刻度，低频左高频右）
  - 纵轴 = 时间（历史滚动）
  - 亮度/色度 = 能量
  - bass 强时整体偏暖色，high 强时偏冷色
- **参数**：滚动速度、透视角度、频带着色、历史深度
- **实现难度**：★☆☆

#### 提案二：波形雕塑（Waveform Sculpture）
- **概念**：将时域波形实时绘制为一条 3D 感的"丝带"，历史帧堆叠形成纵深雕塑。
- **为什么好**：
  - 极简、高级感强，像一件动态艺术品
  - 人声音色不同，波形纹理完全不同，观赏性强
  - 可以配合 Chladni 作为"冷静模式"（Chladni 热闹，波形安静）
- **音频映射**：
  - 当前帧波形 → 丝带截面形状
  - bass → 丝带厚度/振幅放大
  - onset → 爆发性扭曲/颜色闪烁
  - centroid → 丝带表面粗糙度（高频多则锯齿感）
- **参数**：堆叠深度、透视角、振幅增益、扭曲强度、线条粗细
- **实现难度**：★★☆

#### 提案三：引力星座（Audio Gravity）
- **概念**：N 个"恒星"节点分别对应不同频段（bass/mid/high/onset），大量粒子在引力场中运动，音频能量实时改变引力强度和节点位置。
- **为什么好**：
  - 有明确结构（节点是锚点）又有混沌（粒子轨迹不可预测）
  - 和 Dust 同样是粒子系统，但物理意义清晰："能量聚集与爆发"
  - 不同音乐类型产生完全不同的"星系形态"
- **音频映射**：
  - bass 节点：质量大、引力强、移动慢（像黑洞）
  - high 节点：质量小、移动快、轨道偏心率大
  - onset：瞬间增加全局斥力，星系"爆炸"后重组
  - overall：控制背景暗物质密度（影响全局阻尼）
- **参数**：节点数、引力常数、粒子数、暗物质阻尼、爆发阈值
- **实现难度**：★★☆

#### 提案四：环形频谱（Orbital Spectrum）
- **概念**：将频谱数据映射到一个圆环上，每个频段是一个发光的弧段或柱子，粒子沿环流动。
- **为什么好**：
  - 电子乐现场最常见的视觉形式，辨识度高
  - 和 Chladni 的"圆"形成几何呼应，但信息维度完全不同
  - 可以做得非常华丽（发光环 + 粒子尾迹）
- **音频映射**：
  - 环周 = 频率
  - 径向高度/亮度 = 能量
  - bass → 内环脉冲
  - onset → 全环闪光 + 粒子喷射
- **参数**：环数、频段数、粒子流速、发光强度、内环半径
- **实现难度**：★☆☆

## 3. Dust 替代决策

### 3.1 方案对比

| 维度 | 频谱瀑布 | 波形雕塑 | 引力星座 | 环形频谱 |
|------|----------|----------|----------|----------|
| 与 Chladni 差异度 | ★★★★★ | ★★★★☆ | ★★★☆☆ | ★★★★☆ |
| 实现复杂度 | 低 | 中 | 中 | 低 |
| 视觉冲击力 | 中 | 中 | 高 | 高 |
| 电子乐适配 | ★★★★☆ | ★★★☆☆ | ★★★★★ | ★★★★★ |
| 人声乐适配 | ★★★★★ | ★★★★★ | ★★★☆☆ | ★★★☆☆ |
| 独特性 | ★★★☆☆ | ★★★★☆ | ★★★★☆ | ★★★☆☆ |

### 3.2 推荐组合

**方案 A：保守替换（最小开发量）**
- 用「频谱瀑布」替换 Dust
- 保留 Chladni + Liquid
- 理由：频谱瀑布和 Chladni 风格差异最大，一个理性一个感性，且实现极快

**方案 B：激进替换（最大差异化）**
- 用「波形雕塑」替换 Dust
- 扩展 Chladni 增加方形/环形边界
- 把 Liquid 升级为真正的 2D 流体（Navier-Stokes）
- 理由：三个模式各有灵魂——几何物理（Chladni）、时间雕塑（波形）、流体有机（Liquid）

**方案 C：混合替换（平衡）**
- 用「环形频谱」替换 Dust
- Chladni 增加方形边界
- 理由：环形频谱和 Chladni 共享"圆"的几何语言，但一个静态图案一个动态数据，用户切换时不会有割裂感

### 3.3 我的建议

推荐 **方案 B 的变体**：
1. **第一优先**：用「频谱瀑布」替换 Dust（最快获得"不是光尘"的体验）
2. **第二优先**：Chladni 增加「方形板」边界（几行代码，巨大视觉变化）
3. **第三优先**：实现「波形雕塑」作为第四个模式（如果架构允许）
4. **第四优先**：Liquid 的 WebGL 流体（长期项目）

原因：频谱瀑布虽然不算独特，但它**稳定、好看、不犯错**。对于一个已经被 Dust 伤害信任的用户，先给一个"绝对不坏"的选择，比冒险给一个"可能很酷也可能很糟"的新概念更重要。

## 4. 克拉尼多形态的技术细节

### 4.1 方形板（Square Plate）

```javascript
_waveFieldSquare(px, py, m, n) {
  // 将坐标归一化到 [-1, 1]
  const nx = (px - this.cx) / this.maxR;
  const ny = (py - this.cy) / this.maxR;
  // 软边界：超出方形范围后振幅衰减
  const fade = Math.max(0, 1 - Math.max(Math.abs(nx), Math.abs(ny)));
  // 笛卡尔驻波
  let amp = Math.sin(m * Math.PI * (nx + 1) * 0.5) 
          * Math.sin(n * Math.PI * (ny + 1) * 0.5);
  return Math.abs(amp) * fade;
}
```

视觉特征：
- m=1,n=1 → 单拱（像山丘）
- m=2,n=2 → 四宫格
- m=3,n=2 → 水平3条×垂直2条 → 网格交叉
- 节点线是水平和垂直的直线，非常"数字感"

### 4.2 环形板（Annular Plate）

```javascript
_waveFieldAnnular(px, py, m, n) {
  const dx = px - this.cx;
  const dy = py - this.cy;
  const r = Math.sqrt(dx*dx + dy*dy) / this.maxR;
  const theta = Math.atan2(dy, dx);
  const innerR = 0.3; // 内孔半径
  
  if (r < innerR || r > 1.0) return 1.0; // 禁区，amp=1（粒子被推开）
  
  // 径向：在 [innerR, 1] 区间有 m 个半波长
  const rNorm = (r - innerR) / (1 - innerR);
  const radial = Math.sin(m * Math.PI * rNorm);
  const angular = Math.cos(n * theta);
  
  return Math.abs(radial * angular);
}
```

视觉特征：
- 内孔空心，粒子无法进入中心
- m=1,n=3 → 三个弧形段围绕中心空洞
- 像唱片、像瞳孔、像星际环

### 4.3 椭圆板（Elliptical Plate）

椭圆波动方程解析解涉及**马丢函数（Mathieu functions）**，浏览器中无现成库。但可以用数值近似：

```javascript
_waveFieldEllipse(px, py, m, n) {
  // 椭圆坐标变换
  const dx = (px - this.cx) / this.maxR;
  const dy = (py - this.cy) / this.maxR;
  // 简化：用变形极坐标模拟椭圆感
  const a = 1.0;  // 长轴
  const b = 0.6;  // 短轴（偏心率）
  const rEff = Math.sqrt((dx/a)**2 + (dy/b)**2);
  const thetaEff = Math.atan2(dy/b, dx/a);
  
  let amp = Math.sin(m * Math.PI * rEff) * Math.cos(n * thetaEff);
  // 椭圆特有：加入二阶调制产生"猫眼"变形
  amp += 0.3 * Math.sin((m+1) * Math.PI * rEff) * Math.cos((n-1) * thetaEff * 2);
  
  const fade = Math.max(0, 1 - rEff);
  return Math.abs(amp) * fade;
}
```

> 注：这不是严格的椭圆波动方程解，而是一种"视觉近似"。严格解需要预计算或引入 mathjs 等库，考虑到性能，先用近似法验证视觉效果。
